7 
8 Jigsaw Encryption System (JES) takes its inspiration from jigsaw puzzles.
9 Most cryptography methods is a 1-to-1 file method - during encryption, a
10 single unencrypted file (commonly known as plain text) is encrypted into
11 one encrypted file (commonly known as cipher text) using a key.
12 Decryption does the reverse. Mathematically,
13 
14 M{encrypt(plaintext, key) --> ciphertext}
15 
16 M{decrypt(ciphertext, key) --> plaintext}
17 
18 If encryption and decryption are seen as transformation and de-
19 transformation functions, respectively, between plain text and cipher
20 text; the weakest link is in the 1-to-1 file mapping. Brute force attack
21 is possible because all the plain text information exist in the cipher
22 text, though transformed.
23 
24 JES addresses the 1-to-1 file weakness by using 1-to-N file mapping. This
25 is likened to taking a picture and make a jigsaw puzzle out of it. The
26 complexity of brute force attack increases exponentially with the number
27 of jigsaw pieces. By producing many files, it also makes it possible to
28 transport the encrypted file safely via multiple routes, and this prevents
29 reduces the chances of man-in-the-middle attack. This is like making a
30 1000-piece jigsaw puzzle, shuffle the pieces, load the pieces into 5
31 containers and the assembly instruction in the 6th container. If one
32 container is hijacked, all the information is not compromised.
33 
34 Given that the key size of AES-256 is 32 characters long (32 characters
35 of 8 bytes each = 256 bytes) and given that there are 94 characters in
36 the set of mixed alphanumeric with symbols (that is, abcdefghijklmnopqr
37 stuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789!@#$%^&*()-_+=~`[]{}|\:;"'
38 <>,.?/), the total number of keys is 1.38 x 10**63 (94**32).
39 
40 How does this number of permutations compare with Jigsaw Encryption?
41 
42 Assuming that Jigsaw Encryption is used to encrypt a 1 MB (1048576 bytes)
43 file and split into 16 64-KB Jigsaw files, there are 2 x 10**13
44 permutations to assemble 16 64-KB files into the original 1 MB file using
45 Jigsaw version 1.
46 
47 Jigsaw version 2 provides 63 variations for each Jigsaw file. Hence, the
48 full permutations possible for 16 64-KB files is (2 x 10**13)**63 or 9 x
49 10**837, and that is just for a 1 MB file split into 16 64-KB.
50 Jigsaw files.
51 
52 Jigsaw version 3 takes the Jigsaw file provided by Jigsaw version 2 and
53 apply one of the 1350 transpositions. Hence, the full permutations
54 possible for 16 64-KB files is (9 x 10**837)**1350 or 1 x 10**1131238.

122  def swapBlock(self, block, subBlockALocation, subBlockBLocation):
123  '''
124  Function to swap 2 sub-blocks within a block. For example, if the
125  block consist of [1, 'A', 2, 'B', 3]; swapping 'A' and 'B' will
126  result in [1, 'B', 2, 'A', 3].
127 
128  >>> import jigsaw
129  >>> c = jigsaw.JigsawCore()
130  >>> block = 'ABCDEFGHIJKLM'
131  >>> swapped = c.swapBlock(block, (3, 4), (10, 11))
132  >>> swapped
133  'ABCKEFGHIJDLM'
134  >>> reverse_swapped = c.swapBlock(swapped, (3, 4), (10, 11))
135  >>> reverse_swapped
136  'ABCDEFGHIJKLM'
137  >>>
138 
139  @param block: block to be processed.
140  @type block: string
141  @param subBlockALocation: coordinate of first sub-block.
142  @type subBlockALocation: tuple
143  @param subBlockBLocation: coordinate of second sub-block.
144  @type:subBlockBLocation: tuple
145  '''
146  block = [item for item in block]
147  if type(subBlockALocation) == type([]) and \
148  len(subBlockALocation) == 1:
149  subBlockALocation = [0, int(subBlockALocation[0])]
150  elif type(subBlockALocation) == type('str'):
151  subBlockALocation = [0, int(subBlockALocation[0])]
152  if type(subBlockBLocation) == type([]) and \
153  len(subBlockBLocation) == 1:
154  subBlockBLocation = [int(subBlockBLocation[0]), len(block)]
155  elif type(subBlockBLocation) == type('str'):
156  subBlockBLocation = [int(subBlockBLocation[0]), len(block)]
157  preBlockA = block[0:subBlockALocation[0]]
158  BlockA = block[subBlockALocation[0]:subBlockALocation[1]]
159  BlockABlockB = block[subBlockALocation[1]:subBlockBLocation[0]]
160  BlockB = block[subBlockBLocation[0]:subBlockBLocation[1]]
161  postBlockB = block[subBlockBLocation[1]:]
162  swapped = preBlockA + BlockB + BlockABlockB + BlockA + postBlockB
163  return ''.join(swapped)

219  '''
220  Function to take a block, divide the block into smaller
221  sub-blocks, and shuffle the order of the sub-blocks.
222 
223  >>> import jigsaw
224  >>> c = jigsaw.JigsawCore()

230  [3, 1, 2, 0, 4]

236  @type length: integer

239  sblocks = [sb for sb in self.subBlock(block, length)]
